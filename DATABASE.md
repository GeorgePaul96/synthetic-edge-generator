# DATABASE.md

There is **no database**. Persistence is JSON files written by
[CorpusManager](edge_case_engine/corpus.py) plus the generated pytest file, and one in-memory
structure. Read this instead of opening the (generated) data.

## Stores overview

| Store | Location | Lifetime | Written by |
|---|---|---|---|
| Input corpus | `<target_dir>/corpus/inputs.json` | persistent, append-only across runs | `CorpusManager.add_inputs` |
| Crash log | `<target_dir>/corpus/crashes.json` | persistent; **rewritten deduped** at end of CLI run | `record_crash` then `write_deduplicated_crashes` |
| Interesting inputs (seeds) | in-memory `interesting_inputs` | per-run only | `add_interesting_input` |
| Pytest reproducers | `<target_dir>/synthedge_findings.py` | regenerated each CLI run | `PytestExporter.export` |
| Showcase | `examples/corpus/*.json` | committed on purpose (README cites it) | a prior synthedge run |

`<target_dir>` is the directory of the module passed to the CLI; for `python main.py` it is `corpus/`
at the repo root. Serialization: `json.dump(..., indent=2, default=str)`. Dedup key for inputs:
`sha256(json.dumps(value, default=str, sort_keys=True))`.

## `corpus/inputs.json`

JSON **array of input tuples** (each serialized as an array). One entry per unique input generated.

```json
[
  [2.2250738585072014e-308, -9223372036854775809],
  [0.0, -9223372036854775808],
  [-Infinity, 2147483647],
  [NaN, -9223372036854775808]
]
```

- Heterogeneous element types (floats incl. `Infinity`/`NaN`, arbitrary-precision ints, strings,
  `None`, bools). `Infinity`/`NaN` are Python-`json` extensions, not standard JSON.
- Insertion order; unbounded growth across runs. Git-ignored.

## `corpus/crashes.json`

JSON **array of crash records**. During a run, `record_crash` appends raw crashes; at the end the CLI
**overwrites the file with the deduplicated set** (`write_deduplicated_crashes`).

```json
{
  "input":    [<args that triggered the error>],
  "error":    "<ExcType>: <message>",   // e.g. "ValueError: cannot convert float NaN to integer"
  "severity": "HIGH"                     // HIGH = real crash; allowed_exceptions are not recorded
}
```

- `error` format is `"{type(e).__name__}: {e}"` (set in `cli.run_fuzzer`). The exporter and
  deduplicator parse the `ExcType:` prefix from this string.
- `severity` is binary: only crashes (`HIGH`) are recorded; contract-`allowed_exceptions` are skipped
  by the executor and never reach this file. Git-ignored at the repo root.

## In-memory: interesting inputs

Held during a run by `CorpusManager.interesting_inputs`; each entry:

```python
{ "input": <tuple>, "coverage_id": "<sha256 path id | 'seed'>", "energy": <float>, "exec_time_ms": <float> }
```

Consumed by [PowerScheduler](edge_case_engine/scheduler.py) for roulette selection. Lost on exit.

## Generated: `synthedge_findings.py`

A runnable pytest module produced by [PytestExporter](synthedge/exporter.py). One test per
deduplicated+minimized crash:

```python
# Auto-generated by synthedge â€” do not edit manually
import pytest, sys, os
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
from <target_module> import *

def test_synthedge_<fn>_<exc>_000():
    """Found by synthedge: <fn> raises <ExcType> ..."""
    with pytest.raises(<ExcType>):
        <fn>(*<minimized_input>)
```

Floats are rendered nan/inf-safe (`float('nan')`, `float('inf')`). Git-ignored.

## Operational notes

- All generated stores are recreated automatically by running synthedge / `main.py`. Safe to delete
  to reset; `CorpusManager` recreates the `corpus/` dir.
- `examples/corpus/*.json` is the **exception** â€” it is committed showcase data and should be kept.
